HTML

  • HTML: Hypertext Markup Language
  • Elements: Types of content on web page. Examples are, Text elements, Image elements, Header elements, and List elements
  • Head Element contains information or metadata about webpage. Helps browser render page, is not visible when user looks at page
  • Body Element contains content that is shown to user, elements we want user to interact with go here
  • Title: defines the title of the webpage, is required and is important for the SEO
  • SEO or Search Engine Optimization is the process of improving the ranking of a website in the search engines
  • Style: defines CSS styles associated with site
  • Base: defines the base URL for a webpage
  • Link: connects an external resource to an HTML Document
  • Meta: defines meatadata such as character set, description, keywords, author, and viewport
  • Script: embeds client-side scripts in HTML document
  • Charset: element that sets our character encoding
  • http-equiv: sets an HTTP header for content
  • Viewport: sets the default view for a webpage
  • Child Elements are elements nested inside other elements, Parent elements are the ones they are nested inside
  • In Seantic HTML Elements the name of the element describes its content and the role it plays on the web page
  • div: creates a division in the page
  • span: creates inline division
  • On visible HTML elements all content is between an opening and closing tag
  • Element and Tag are commonly used interchangably, there are differences though
  • Element- entire element from opening tag to closing tag
  • Tag- refers to only what is inside angle brackets
  • p: Represents a paragraph or block of text
  • ul: unordered list
  • ol: ordered list
  • li: list item
  • img: contains info about images displayed
  • src= attribute most important because it deffines the location of image
  • br: creates a line of empty space, or line break, between two blocks of content
  • a (stands for anchor): creates link to the same webpage or other webpages
  • href: attribute points to the URL for the link
  • id: attribute is a way of identifying elements and selecting them

CSS

  • Selector defines the element or attribute to which the rules/declaration will aply
  • Declarations contain two conponents, the CSS property we want to apply and the value of the property
  • Margin property acts like a margin in a word processing document, it indicates how much space we want on the OUTSIDE of an element, not visible just acts like a buffer
  • Padding property adds space around the content INSIDE an element, increase in padding makes entire element bigger
  • To apply a rule to all elements on a page use special wild card symbol or *
  • Combining two selectors into a single CSS rule is done using a comma, helps not write "Do Not Repeat Yourself" (Dry) Code
  • DRY is a principle that encourages devs to share or reuse code in order to reduce lines of code
  • rel attribute specifies relationship between current document and the linked document resource
  • href attribute specifiesthe location (URL) of external resource
  • Class attribute allows us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector
  • Assigning colors in CSS can be done by semantic references, hexadecimal code, or hex #

Git

  • git status: checks what branch we are currently in
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull: fetch and merge any commits from the tracking remote branch
  • git push [alias]/[branch]: transmit local branch to the remote repository branch
  • git commit -m "": commit your staged content as a new commit snapshot
  • git merge [branch]: merge the specified branche's history into your working directory
  • git log: shows all commits in the current branches history
  • Organization: a way of grouping and allowing teams to collaborate across many projects at once
  • Repository (repo): a folder in which all files and their version histories are stored
  • Branch: a version of the repo that allows work without affecting other branches. Repos may have many branches for different possible changes being tested or considered, along with a default branch that serves as the source of truth
  • Fork: a new repository that inherits from a parent “upstream” repo. It is used to suggest changes to an “upstream” public repo by someone who doesn’t have access to edit in the repo’s home org
  • Markdown (.md):a way to write content that converts plain text to formatted text.
  • Commit Changes: a saved record of a change made to a file within the repo
  • Pull Request (PR): a request for changes made to a branch to be pulled into another branch. Allows multiple users to see, discuss and review work being suggested
  • Merge: after a pull request is approved, the commit will be pulled in (or merged) from one branch to another and then, deployed on the live site
  • Issues: allow users to report issues or bugs and track progress of assigning the fix for the issues
  • Federalist: a platform that securely deploys a website from a GitHub repository in minutes and lets users preview proposed and published changes
  • Projects: allows you to use GitHub for project management and tracking a set of issues, either for a specific repo or an entire org
  • Wiki — a section of a repo made for hosting documentation. Documentation may be in the repo’s README files instead

Javascript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • Browser Application Programing Interfaces (APIs) built into web browsers providing functionality such as dynamically creating HTML and setting CSS styles; collecting and manipulating a video stream from a users webcam, or generating 3D graphics and audio samples
  • Good Practice to have semicolon at the end of each statement
  • JavaScript is case sensitive
  • Can retrieve value by calling variable name, can change value later in code
  • String: Sequence of text, to signify enclose it with single or double quotes
  • Number: just a number, does not have quotes
  • Boolean: true or false value, special keywords that do not need quotes
  • Array: Structure that allows you to store multiple values in a single reference
  • object: can be anything, everything in Java is an object and can be stored as a value
  • Comments: Snippets of text added in code, browser ignored this text
  • Operator: Mathmateical symbol that provides a result based on two values or variables
  • Addition:+: adds two numbers together or combines strings
  • Subtraction:-, Multiplication:*, Division:/; all do as you expect in math
  • Assignment:=: this assigns a value to a variable
  • Strict Equality:===: this performs a test to see if two values are equal and of the same data type
  • Not, Does-not-equal:!,!==: returns the logically opposite value of what it precedes. Turns true into false. When used alongside the Equality Operator, the negation operator tests whether two values are not equal
  • Conditionals: Code structures used to test if an expression returns true or not
  • Functions: way of packaging functionality that you wish to reuse
  • Argumnents: data functions need to do their job, go inside ()
  • Primitive Data: used to store simple pieces of data
  • Null Type: inhabited by one value - null, indicates absense of an object
  • Undefined type: inhabited by one value - undefined, indicates the absense of a value
  • Boolean Type: true or false, used for conditional operations
  • Number type: a double-precision 64-bit binary format IEEE 754 value
  • NaN: Not a Number: a kind of number that is encountered when the result of an arithmetic operation can't be expressed as a number, only value in Java not equal to it self
  • BigInt: a numeric primitive that can represent integers with arbitary magnitude
  • String: type represents textual data and is encoded as a sequence of 16-bit unsigned integer values representing UTF-16 code units
  • JavaScript Strings are immutable, meaning once a string is created it cannot be changed
  • Use strings for textual data. When representing complex data, PARSE STRINGS, and use the appropriate abstraction
  • Symbol Type: unique and immutable primitive value and may be used as the key of an object property
  • Object: a value in memory which is possibly referenced by an integer
  • Data Property: associate a key with a value
  • Attributes that describe it:
  • Value- value by a get access of the property
  • Writable- boolean value indicating if the property can be changed with an assginment
  • Enumerable- boolean value indicating if the property can be enumerated by a "for...in" loop
  • Configurable- Boolean value indicating if the property can ne deleted, can be changed to an assessor property, can have its attributes changes
  • Accesor Property: associates a key with one of two accessor functions (get and set) to retrieve or store a value
  • Has the following attributes:
  • get- function called with an empty argumnent list to retrieve the property value whenever a get access to the value is preformed, may be undefined
  • set- function called with an argumnentthat contains the assigned value, executed whenever a specific property is attempted to be changed, may be undefined
  • Enumerable- see above
  • Configurable- see above, can be changed to a data property
  • When representing dates, best choice is to use built-in "Date" utility
  • Control Flow - order in which a computer executes code in a script
  • Arrays: regular objects for which there is a particular relationship between integer-keyed properties and the length property
  • Typed Arrays: present an array-like view of an underlying binary data buffer, and offer many methods that have similar semantics to the array counterparts
  • Set and WeakSet represent a collection of unique values, while Map and WeakMap represent a collection of key-value associations
  • JSON (JavaScript Object Notation) is a lightweight data-interchange format, derived from JavaScript, but used by many programming languages
  • PRIMITIVE COERCION process is used where a primitive value is expected, but there's no strong preference for what the actual type should be
  • Numeric coercion- the automatic conversion of values from one data type to another
  • "For" Loops: 1- keyword "for", 2- Three part conditional statement, 3- code block
